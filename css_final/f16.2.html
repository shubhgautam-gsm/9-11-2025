Width vs. Flex-Basis: Seeing the Difference

Scenario 2: Using `flex-basis` with `flex-shrink: 0`

Container width: 300px. Items have `flex-grow: 0` and `flex-shrink: 0`.

Desired total flex-basis: 100px (A) + 100px (B) + 150px (C) + 30px (margins) = 380px

Similar to `width`, since `flex-shrink: 0` prevents shrinking, **item C will also overflow** the container.

Scenario 3: `flex-basis` vs `width` when both are present (and `flex-basis` wins)

This explicitly shows that `flex-basis` takes precedence over `width` for the *initial* size of a flex item.

X (width:50, flex-basis:120)
Y (width:150, flex-basis:80)
Z (width:70, flex-basis:100)

Notice how items X, Y, and Z size themselves according to their `flex-basis` values, ignoring their `width` values for the initial sizing.
